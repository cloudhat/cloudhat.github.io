# Essay Writing

에세이, 회고, 커리어 기록 등 개인 글을 글쓰기 원칙에 따라 진단하고 개선한다.

인자: `$ARGUMENTS` (예: `_posts/.../파일.md --review` 또는 `--rewrite <섹션명>`)

## 글쓰기 원칙

### A. 도입과 결론

**도입 — 장면으로 시작하라.**
가장 강력한 오프닝은 구체적인 순간에 독자를 던져넣는 것이다. 배경 설명, 사전 정의, 넓은 일반론으로 시작하지 마라.

Hook 유형:
- Anecdote: 특정 순간의 장면
- 충격적 사실: 수치, 반전
- 질문: 독자가 생각하게 만드는 질문

참고: Zinsser *On Writing Well* — 도입부에 "인간적 요소"를 넣어라.

**결론 — 요약이 아니라 울림이다.**
- 원점 회귀(Callback): 도입부 일화/이미지를 결론에서 새로운 시각으로 회수한다.
- 마지막 문장은 짧고, 강하고, 리듬감 있게 끝낸다.
- 피할 것: "지금까지 ~에 대해 살펴보았다"식 기계적 요약, 본문에 없던 새 논점 도입.

참고: Zinsser — "The last sentence is as important as the first."

### B. 내용 전개

**Show → 성찰 순서.** 감각적 묘사(Show)만 있으면 장면으로 끝난다. 반드시 Show 다음 성찰 한 문장을 붙인다.

참고: Philip Lopate — 개인 에세이는 묘사와 성찰의 교차가 핵심이다.

**구체 → 추상 순서.** 숫자, 장면, 사례가 먼저다. 추상적 교훈은 마지막에 한 번만 쓴다.

**문단 = 하나의 논점.** 각 문단은 핵심 주장(topic sentence)으로 시작하고, 구체 사례로 뒷받침한다. 하나의 문단에 감정/논점이 둘 이상 들어가면 쪼갠다.

참고: Strunk & White *The Elements of Style* — "Make the paragraph the unit of composition."

**"So what?" 테스트.** 각 문단에 "그래서 뭐?"라고 물었을 때 답이 없으면 삭제하거나 보강한다.

### C. 흐름과 전환

**구정보-신정보 계약(Old-New Contract).** 각 문장의 앞에는 이미 알려진 정보를, 뒤에는 새 정보를 배치한다. 접속어 없이도 흐름이 자연스러워진다.

참고: Joseph Williams *Style: Lessons in Clarity and Grace*

**핵심어 반복(Echo threading).** 이전 단락의 핵심 개념이나 단어를 다음 단락 첫 문장에 자연스럽게 반복하여 연결감을 준다.

**문단 끝은 다리(bridge)다.** 문단 마지막 문장이 다음 문단으로 넘어가는 연결을 담당한다. 전환 없이 끝나면 독자가 이탈한다.

체크리스트:
- [ ] 문단 첫 문장만 읽어도 핵심을 알 수 있는가?
- [ ] 마지막 문장이 다음 문단으로 연결되는가?

### D. 문장 기술

**군더더기 제거(Orwell 원칙).**
- 짧은 단어로 충분하면 긴 단어를 쓰지 마라
- 단어를 뺄 수 있으면 반드시 빼라
- 능동태를 쓸 수 있으면 수동태를 쓰지 마라
- 진부한 비유/은유를 쓰지 마라

참고: George Orwell *"Politics and the English Language"* (1946)

**강한 동사를 써라.** 명사화(nominalization)를 피한다. "검토를 실시하다" → "검토하다". 약한 동사 + 명사 조합을 구체적 동사 하나로 바꾼다.

참고: Williams *Style* — 명사화를 동사로 되돌리면 문장이 산다.

**문장 길이를 변주하라.** 짧은 문장과 긴 문장을 섞는다. 짧은 문장은 강조, 긴 문장은 설명. 단조로운 리듬은 독자를 잠재운다.

참고: Gary Provost — "This sentence has five words..."

### E. 퇴고

**단계적 퇴고.** 한 번에 모든 것을 고치지 않는다. 회차별로 초점을 달리한다.
- 1차: 구조와 논리 흐름 (단락 순서 재배치, 불필요한 단락 삭제)
- 2차: 문장 수준 다듬기 (간결성, 리듬, 어색한 표현)
- 3차: 표면적 교정 (오탈자, 맞춤법, 문장부호)

참고: Richard Lanham *Revising Prose* — Paramedic Method

**10% 규칙.** 초고에서 최소 10%를 삭제하라. 거의 모든 글은 줄일수록 좋아진다.

참고: Stephen King *On Writing* — "2nd Draft = 1st Draft − 10%"

**소리 내어 읽어라.** 어색한 리듬, 반복, 호흡이 끊기는 문장을 귀로 잡아낸다. 눈으로는 놓치는 문제를 소리가 드러낸다.

## 실행 절차

`$ARGUMENTS`를 파싱한다:
- 파일 경로만 있거나 `--review`: 진단 리포트 모드
- `--rewrite <섹션명>`: 해당 섹션 초안 제시 모드

### `--review` (기본): 진단 리포트

1. 파일 Read
2. 섹션별로 글쓰기 원칙(A~E) 체크
3. 문제 부분을 원문 인용 + 이유 + 개선 방향으로 리포트 출력

리포트 형식:
```
## [원칙명] — 섹션명

> 원문 인용

**문제**: 왜 어색한지
**개선 방향**: 어떻게 바꿀지
```

### `--rewrite <섹션명>`: 초안 제시

1. 해당 섹션 Read
2. 글쓰기 원칙(A~E) 적용하여 초안 작성
3. 원문과 나란히 제시, 변경 이유 설명
4. 사용자 확인 후 Edit

## 주의사항

- 내용(팩트, 경험)은 수정하지 않는다. 문장 구조와 흐름만 다듬는다.
- 필자의 목소리(tone)를 유지한다. 문어체/구어체 혼합 스타일을 임의로 통일하지 않는다.
- 초안 제시 후 반드시 사용자 확인을 받고 Edit한다.
