# 블로그 글쓰기 스타일 가이드

이 문서는 기존 블로그 글 18편을 분석하여 추출한 글쓰기 스타일 가이드입니다.
LLM에게 글 작성을 요청할 때 이 문서를 함께 전달하세요.

---

## 1. Front Matter

```yaml
---
title: 한글 제목
date: YYYY-MM-DD HH:MM:SS +0800
categories: [대분류, 소분류]
tags: [태그1, 태그2, ...]
---
```

- 타임존은 항상 `+0800`
- 카테고리 예시: `[Backend, Architecture]`, `[Infra, Docker]`, `[Retrospective, Develop]`, `[PM, Agile]`

## 2. 문체와 톤

### 기술 글: 존댓말 (`~습니다`, `~합니다`)
> "다소 부끄러운 이야기지만 당시 신입이었던 저는 동시성 문제를 알지 못한 채 선착순 로직을 구현했습니다."

### 에세이/회고 글: 반말 (`~한다`, `~했다`, `~이다`)
> "많은 직장인들이 오늘도 퇴근 후에 치열하게 자기계발을 한다."

### 톤 특징
- **솔직하고 자기고백적**: 과거 실수를 숨기지 않음 ("다소 부끄러운 이야기지만", "이는 큰 오산이었습니다")
- **열정적이고 진지함**: 개발에 대한 확신 ("이런 고민을 해결하는 과정이 전혀 괴롭지 않고 오히려 즐겁습니다")
- **실무 기반**: 항상 회사에서의 실제 경험에 근거 ("저희 회사에서는...")
- **겸손하지만 자신감 있음**: 한계를 인정하면서도 성장을 보여줌
- **Trade-off 의식**: 솔루션을 완벽하다고 제시하지 않음, 항상 장단점 분석

## 3. 글 구조

### 기술 글 템플릿

```
## 개요
- 이 글에서 다루는 내용을 bullet point로 요약

<br>

## 배경
- 회사에서 발생한 실제 상황 설명
- "저희 회사에서는..." 으로 시작

<br>

## 문제상황
- 구체적인 문제 기술 (수치 포함: "CPU 100%", "초당 1만건" 등)

<br>

## 해결 (또는 구현, 도입)
### 1) 첫 번째 접근
### 2) 두 번째 접근
- 코드 블록은 **볼드 라벨**로 설명 후 삽입
- Before/After 패턴 활용

<br>

## 장점
## 단점 (또는 한계)

<br>

## 마치며 (또는 결과)
- 개인적 성장 회고
- 미래 목표

<br>

## 참고한 자료
- 링크 목록
```

### 핵심 규칙
- 섹션 사이에 `<br>` 태그로 시각적 간격 부여
- 최상위 섹션은 `##`, 하위 섹션은 `### 1)`, `### 2)` (닫는 괄호만 사용)
- 더 깊은 하위는 `**(1)**, **(2)**` 또는 `**a.**, **b.**`

## 4. 도입부 패턴

### 패턴 A: Bullet point 개요 (기술 글)
```markdown
## 개요
- Redisson을 이용한 Distributed Lock의 도입 근거
- **Redis에 재고를 선반영**하는 선착순 쿠폰 지급 로직 소개
- **Redis에 재고를 선반영**하는 로직의 한계 및 대안 제시
```

### 패턴 B: 실무 배경 서술
```markdown
저희 회사에서는 소비자들의 만족도와 구매촉진을 위해 다양한 이벤트를 제공하고 있습니다.
```

### 패턴 C: 철학적 질문 (에세이)
```markdown
## 왜 자기계발을 하는가?
많은 직장인들이 오늘도 퇴근 후에 치열하게 자기계발을 한다.
```

## 5. 마무리 패턴

### 성장 회고 (가장 빈번)
> "불과 1년 전만 해도 저는 '동시성 문제'라는 키워드 조차 모르는 부족한 개발자였습니다. 그러나... 많은 경험을 얻었습니다."

### 미래 목표
> "- 테스트 코드를 추가하여 서비스의 안정성을 높일 예정입니다"

### 피드백 요청
> "최선을 다했지만 아직 부족한 점이 많습니다. 피드백은 언제든지 환영입니다."

### 시그니처 표현
> "코딩이란 낭만입니다. 과정 자체가 즐겁기에 낙관과 함께 성장하고 있습니다."

### 시니어 개발자 열망
> "저는 이런 경험이 켜켜이 쌓여 언젠가 훌륭한 시니어 개발자가 되고 싶다는 간절한 열망이 있습니다."

## 6. 코드 블록 사용

- 언어: 주로 `java`, 필요시 `sql`, `yaml`, `json`
- 코드 앞에 반드시 **볼드 라벨**로 설명:
  ```
  **(1) 어노테이션을 위한 클래스 선언**
  ```
- 생략 표시: `//생략` 또는 `.` dots
- 주석은 한국어 위주: `//레디스에서 '현재 발급된 쿠폰 수' 조회`
- 개념 설명용 의사코드에서는 한국어 식별자 사용 가능:
  ```java
  public class 업무요청 {
      public void 단기_업무요청_생성(){
  ```

## 7. 시각적 요소

### 이미지
- 경로: `/assets/img/posts/YYYY/YYYY-MM-DD-slug/image-name.png`
- 형식: `![alt](/path)`

### 강조
- **볼드** (`**...**`): 핵심 개념 강조에 적극 활용
- ~~취소선~~ (`~~...~~`): 업데이트된 내용, 더 이상 유효하지 않은 정보
- <ins>밑줄</ins> (`<ins>...</ins>`): 매우 중요한 강조

### 인용 블록
```markdown
> **참고자료**
> - 링크1
> - 링크2
```

### 접이식 콘텐츠
```html
<details>
<summary>제목</summary>
<div markdown="1">
내용
</div>
</details>
```

### 사후 업데이트 표기
- `*{25.09} 추가 내용...`
- `[ *추가내용(25-08) : ...]`
- `(*25.10 추가) 내용...`

## 10. 금기 사항 (하지 않는 것들)

- 순수 이론만 다루지 않음 → 반드시 실무 경험 기반
- 솔루션을 완벽하다고 제시하지 않음 → 항상 한계/단점 언급
- 과도하게 격식 차리지 않음 → 적당히 친근하고 솔직
- 영어로만 된 전문용어를 남발하지 않음 → 한국어 번역 병기 (예: "결과적 일관성(Eventual Consistency)")
- 감정 과잉 표현 사용 안 함 → 절제된 열정

## 11. 시리즈 글 형식

시리즈물은 글 상단에 목차 링크 제공:

```markdown
## 시리즈명
- 1부 : [제목](링크)
- **2부 : 현재 글 제목** ← 현재 글은 볼드
- 3부 : [제목](링크)
```

## 12. 자기 소개 키워드

글쓴이의 정체성을 반영하는 핵심 키워드:
- 성장하는 개발자
- 실무에서 배우는 사람
- 기술부채와 싸우는 사람
- "코딩은 낭만이다"
- 훌륭한 시니어 개발자가 되고 싶은 열망
